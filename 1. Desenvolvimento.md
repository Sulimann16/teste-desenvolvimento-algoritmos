# 🎢 PROJETO: Totem de Autoatendimento "MegaloDon"

**Disciplina:** Desenvolvimento de Algoritmos  
**Atividade:** Desenvolvimento de Software com Foco em Requisitos

---

## 📋 Contexto do Projeto
O parque de diversões *TechPark* está automatizando a entrada da sua atração principal, a montanha-russa **MegaloDon**. Você foi contratado para desenvolver o algoritmo do totem de autoatendimento que valida a entrada dos clientes na fila. 

O sistema precisa avaliar as condições físicas do cliente, emitir alertas personalizados e garantir que ninguém corra riscos de segurança. 

**Atenção:** Escreva o código mais limpo, organizado e claro possível, pois, ao final do tempo regulamentar, seu algoritmo será sorteado e enviado para um colega fazer o *Code Review* (revisão de código).

---

## 🛠️ Requisitos do Sistema (As Regras do Software)

Seu programa deve interagir com o usuário solicitando três informações, uma por uma. Para cada informação, o sistema deve aplicar regras rígidas de validação antes de avançar.

### 1. Entrada e Validação: Nome do Cliente
* **Tipo de dado esperado:** Texto (`String`).
* **Regra de Validação:** O sistema deve verificar se o campo não foi deixado em branco. Se o usuário pressionar apenas "Enter" ou digitar um texto vazio `""`, o sistema deve exibir a mensagem abaixo e encerrar o programa imediatamente:
  > `"[ERRO] O nome do cliente não pode estar vazio!"`

### 2. Entrada e Validação: Idade do Cliente
* **Tipo de dado esperado:** Número Inteiro (`Inteiro`).
* **Regra de Validação:** Se a idade digitada for **menor ou igual a 0 (zero)** OU se for **maior que 120**, o sistema deve exibir a mensagem abaixo e encerrar o programa imediatamente:
  > `"[ERRO] Idade inválida! Digite um valor entre 1 e 120."`

### 3. Entrada e Validação: Altura do Cliente
* **Tipo de dado esperado:** Número Decimal/Real (`Real` / `Float`). *Atenção para receber o valor em metros (ex: 1.75).*
* **Regra de Validação:** Se a altura digitada for **menor ou igual a 0.50m (50cm)** OU se for **maior que 2.50m**, o sistema deve exibir a mensagem abaixo e encerrar o programa imediatamente:
  > `"[ERRO] Altura inválida! A altura deve estar entre 0.50m e 2.50m."`

---

## 🏗️ Regras de Negócio (Critérios de Acesso)

Se todos os dados passarem nos testes de validação acima, o algoritmo deve processar as regras de segurança do brinquedo. 

* **Regra de Ouro da MegaloDon:** Para entrar, o cliente precisa ter **no mínimo 12 anos** e **no mínimo 1.50m de altura**.

O sistema deve analisar a combinação dos dados e exibir **apenas uma** das quatro mensagens personalizadas abaixo:

1. **Acesso Permitido (Sucesso):**
   * *Critério:* Cumpriu a idade mínima e a altura mínima.
   * *Mensagem:* `"Olá, [Nome]! Cadastro validado com sucesso. Seu acesso foi PERMITIDO. Divirta-se na MegaloDon!"`
2. **Bloqueio apenas por Idade:**
   * *Critério:* Tem a altura necessária (>= 1.50), mas não tem a idade necessária (< 12).
   * *Mensagem:* `"[ACESSO NEGADO] - [Nome], você possui a altura necessária, mas não atingiu a idade mínima de 12 anos."`
3. **Bloqueio apenas por Altura:**
   * *Critério:* Tem a idade necessária (>= 12), mas não tem a altura necessária (< 1.50).
   * *Mensagem:* `"[ACESSO NEGADO] - [Nome], você possui a idade necessária, mas não atingiu a altura mínima de 1.50m."`
4. **Bloqueio por Ambos os Motivos:**
   * *Critério:* Não tem nem a idade (< 12) e nem a altura (< 1.50).
   * *Mensagem:* `"[ACESSO NEGADO] - [Nome], o acesso foi recusado porque você não possui a idade mínima (12 anos) e nem a altura mínima (1.50m)."`

## 💡 Dicas Importantes para o Desenvolvimento
1. **Ordem de Execução:** As validações de erro (dados vazios, números negativos ou absurdos) devem acontecer **antes** de testar se a pessoa pode ou não entrar no brinquedo. Se o programa detectar um erro na idade, ele não deve nem chegar a pedir a altura.
2. **Identação (Recuo de Código):** Alinhe corretamente os blocos de `Se/Senão` (`if/else`). Código bagunçado será penalizado pelo colega na etapa de revisão.
