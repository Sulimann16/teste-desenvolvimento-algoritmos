# 🔍 GUIA DE CODE REVIEW e VALIDAÇÃO DE TESTES

**Papel do Aluno:** Engenheiro de Qualidade (QA) / Revisor de Código  
**Objetivo:** Avaliar o algoritmo desenvolvido pelo seu colega, testar a robustez do sistema e fornecer feedback construtivo.

---

## 📋 Como Proceder
1. Abra o código do seu colega no ambiente de desenvolvimento (ou faça o teste de mesa no papel, linha por linha).
2. Execute o programa **7 vezes**, inserindo em cada execução exatamente os dados descritos na tabela de testes abaixo.
3. Verifique se a resposta que aparece na tela é **idêntica** ao resultado esperado. Se o programa fechar com erro inesperado, travar ou mostrar a mensagem errada, anote como um **Bug**.

## 🔎 Checklist de Análise de Código
Além de rodar os testes, olhe para as linhas de código do colega e responda:

* [ ] **Clareza de Variáveis:** O colega usou nomes claros (ex: `idade`, `altura`, `nome`) ou usou nomes confusos que não explicam nada (ex: `x`, `y`, `a`, `b`)?
* [ ] **Identação:** O código está alinhado e fácil de ler? Dá para entender visualmente o que está dentro de cada bloco `Se/Senão`?
* [ ] **Otimização:** A lógica usada nas estruturas condicionais foi direta ou ele deu voltas desnecessárias para chegar ao resultado?

---

## 💬 Roteiro de Feedback (Sessão de Alinhamento)
Ao finalizar os testes, junte-se com o dono do código por 5 minutos para entregar o resultado. Seja profissional e utilize o roteiro abaixo:

1. **Elogio Técnico:** Destaque uma linha de código, comentário ou solução lógica que seu colega fez muito bem.
2. **Apontamento de Bugs:** Mostre quais testes da tabela falharam (se houver) e ajude-o a entender o porquê.
3. **Sugestão de Melhoria:** Diga uma coisa que tornaria o código dele mais legível ou elegante (ex: "Se você mudar o nome dessa variável, quem ler seu código no futuro vai entender mais rápido").
